# Engine Version Split: source-built engine vs UE 5.7

> **Status:** active  
> **Owner:** UnrealMCP plugin maintainers  
> **Last updated:** 2026-05-14

This document explains why the repository ships **two copies** of the
`UnrealMCP` Unreal plugin and how to keep them in sync.

---

## 1. Why two copies?

The repository hosts **two Unreal projects**:

| Project | Engine | `EngineAssociation` | Purpose |
|---|---|---|---|
| `FlopperamUnrealMCP/` | source-built engine (custom branch) | `""` | engine team / runtime work |
| `FlopperamUnrealMCP 5.7/` | UE 5.7 (marketplace launcher build) | `"5.7"` | gameplay/content/M2 work |

Historically both projects pointed at the canonical plugin via
`AdditionalPluginDirectories: ["../Plugins"]` in their `.uproject` files.
That setup forces UnrealBuildTool to **invalidate the plugin's
`Intermediate/` cache every time you switch engines**, because each engine
expects its own UBT module stamps. The result was a full plugin rebuild
(several minutes) on every project switch - tracked as **issue #2**.

To eliminate this rebuild churn:

- The **canonical source of truth** stays at the repo root: `Plugins/UnrealMCP/`.
- The **UE 5.7 project** continues to consume the canonical copy via
  `AdditionalPluginDirectories: ["../Plugins"]` (unchanged).
- The **source-built project** now keeps its own project-local copy at
  `FlopperamUnrealMCP/Plugins/UnrealMCP/`. Its `.uproject` no longer sets
  `AdditionalPluginDirectories`, so UBT keeps its `Intermediate/` and
  `Binaries/` strictly inside `FlopperamUnrealMCP/Plugins/UnrealMCP/`.

```
unreal-engine-mcp/
├── Plugins/
│   └── UnrealMCP/                              <-- canonical (UE 5.7 reads here)
│       ├── UnrealMCP.uplugin
│       ├── Source/
│       └── Config/
├── FlopperamUnrealMCP/                         <-- source-built project
│   └── Plugins/
│       └── UnrealMCP/                          <-- project-local copy
│           ├── UnrealMCP.uplugin               (generated by sync script; git-ignored)
│           ├── Source/
│           └── Config/
└── "FlopperamUnrealMCP 5.7"/                   <-- UE 5.7 project
    └── (no local Plugins/UnrealMCP)            (uses ../Plugins via .uproject)
```

## 2. Where do I edit the plugin?

**Always edit the canonical copy** at `Plugins/UnrealMCP/`. This is the
single source of truth and the only path that the UE 5.7 project sees.

Then run the sync script to push your edits into the source-built project:

```powershell
.\scripts\sync-unrealmcp-plugin.ps1
```

Reverse-direction sync is supported when you need to promote an emergency
edit made inside `FlopperamUnrealMCP/Plugins/UnrealMCP/` back to canonical:

```powershell
.\scripts\sync-unrealmcp-plugin.ps1 -Reverse
```

> ⚠️ **Reverse sync overwrites the canonical copy.** Use it only when the
> source-built copy is newer than canonical, and double-check `git diff`
> before committing.

## 3. CI / drift detection

Run the script in `-Verify` mode to fail fast if the two copies diverge.
This is the recommended CI guardrail before any plugin-touching PR is
merged.

```powershell
.\scripts\sync-unrealmcp-plugin.ps1 -Verify
```

Exit codes:

| Code | Meaning |
|---|---|
| 0 | source and target match |
| 2 | drift detected (script also prints the offending file list) |

---

## 4. What the sync script does NOT copy

The script intentionally excludes the following directories so build
outputs stay scoped to the project that produced them:

- `Binaries/`
- `Build/`
- `DerivedDataCache/`
- `Intermediate/`
- `Saved/`
- `.vs/`, `.vsconfig`

The repo's top-level `.gitignore` already excludes these patterns
(`**/Binaries/`, `**/Intermediate/`, etc.), so the project-local copy at
`FlopperamUnrealMCP/Plugins/UnrealMCP/` will never accidentally check in
build artifacts.

## 5. Common workflows

### 5.1 Edit the plugin (typical case)

```powershell
# 1. Edit Plugins/UnrealMCP/Source/... in your IDE.
# 2. Sync into the source-built project copy.
.\scripts\sync-unrealmcp-plugin.ps1
# 3. Open EITHER project and rebuild.
```

### 5.2 Verify a clean checkout is in sync

```powershell
.\scripts\sync-unrealmcp-plugin.ps1 -Verify
```

### 5.3 Initial setup of a fresh clone

The source-built copy is generated and ignored by git. On a fresh clone, run
the sync script once before opening `FlopperamUnrealMCP/`:

```powershell
.\scripts\sync-unrealmcp-plugin.ps1
.\scripts\sync-unrealmcp-plugin.ps1 -Verify
```

## 7. FAQ

**Q. Why not symlinks?**  
A. `mklink /D` requires either Developer Mode or admin rights on Windows
and breaks UGS / Perforce workflows. A copy + sync script is portable and
auditable.

**Q. Why isn't the source-built project the canonical source?**  
A. The UE 5.7 project is the active M2 build and gets edited far more
frequently. Pointing it at the root is a one-line `.uproject` setting
(`AdditionalPluginDirectories`); pointing the source-built project at the
root re-introduces the rebuild churn we are trying to remove. Keeping the
canonical copy at the root means the UE 5.7 project never has to re-stamp
`Intermediate/`.

**Q. What if I forget to sync?**  
A. The source-built project will keep building against an older copy of
the plugin. Run `./scripts/sync-unrealmcp-plugin.ps1` and then `-Verify`
before testing or packaging the source-built project.

**Q. Do I need to sync after every edit?**  
A. Only when you intend to test in the source-built project. The UE 5.7
project always sees the canonical copy directly.
